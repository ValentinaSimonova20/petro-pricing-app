Кардинальность в нотации Crow’s foot. PK отмечены в блоках сущностей.

PK — первичный / логический ключ FK — поле связи линия — join по именам колонок Excel

Имена полей на диаграмме слегка нормализованы (пробелы → _) ради читаемости Mermaid; канонические имена — в data-model.md и на вкладке «Поля».

Связи join-по-join

Точные условия соединения колонок — как в data-model.md.

От К Условие Кардинальность
formulas formula_components Ключ формулы = Формула 1 : N (полное покрытие)
formula_components term_types Тип фиксированного терма = type_code N : 1
formulas ssp client_id = client_id N : M (по клиенту)
formulas material_groups Подгруппа материалов = hname N : 1 (покрытие неполное)
formulas material_groups / ssp Материал ↔ code_nsi / mtr_nsi_code N : M
ssp material_groups mtr_nsi_code (как строка) = code_nsi N : M
formula_components quote_mapping Имя котировки (+ при необходимости Происхождение котировки) N : 0..1
quote_mapping quotes ID в озере данных = quote_code 1 : N
formula_components currency_rates тип терма 5 → currency_name + calday логический join
