# Модель данных (этап 1)

Документ описывает сущности первого этапа разработки — исходные справочники и таблицы, на которых строится подбор формулы и расчёт цены. Источники MVP: восемь Excel-файлов (по одному листу в каждом). Имена полей приведены к рабочим идентификаторам из выгрузок; где в Excel русские заголовки — указаны оба варианта.

Типы данных — логические (для проектирования схемы БД / ORM); конкретная СУБД на этом этапе не фиксируется.

---

## Обзор сущностей

| Сущность | Источник | Назначение | Первичный ключ |
| --- | --- | --- | --- |
| Строка прогноза спроса | `ssp.xlsx` | Позиция «клиент + материал + период», для которой нужно посчитать цену | `row_id` |
| Формула | `formulas.xlsx` | Договорное выражение цены и область применения | `formula_id` |
| Компонент формулы | `formula_components.xlsx` | Переменная выражения: тип, значение или ссылка на котировку/курс | (`formula_id`, `var_name`) |
| Тип терма | `term_types.xlsx` | Расшифровка кодов типа компонента SAP | `type_code` |
| Маппинг котировок | `quote_mapping.xlsx` | Соответствие SAP-имени котировки и кода в озере данных | `quote_tech_name` (имя котировки) |
| Котировка | `quotes.xlsx` | Рыночная цена на дату | (`quote_code`, `quote_date`, `quote_type`) |
| Курс валюты | `currency_rates.xlsx` | Курс валютной пары на день | (`currency_name`, `calday`, `version_type`) |
| Группа материалов | `material_groups.xlsx` | Состав группы M: какие `material_id` входят в группу | (`group_m`, `material_id`) |

Логические объекты **Клиент** и **Материал** в отдельных файлах не выгружены: их атрибуты денормализованы в прогнозе спроса и группах материалов (`client_id` / `client_name`, `material_id` / `material_name`).

---

## 1. Строка прогноза спроса (`DemandForecastLine`)

Одна строка — одна позиция планирования. По ней сервис ищет формулу и считает цену.

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| `row_id` | `integer` | да | Уникальный идентификатор строки |
| `period` | `date` | да | Период прогноза (месяц; в выгрузке — дата, напр. `2027-12-01`) |
| `client_id` | `string` | да | Идентификатор клиента (обезличенный, напр. `CL-10143`) |
| `client_name` | `string` | нет | Отображаемое имя клиента |
| `material_id` | `string` | да | Идентификатор материала (SKU) |
| `material_name` | `string` | нет | Наименование материала |
| `forecast_tons` | `decimal` \| `null` | нет | Прогнозный объём, тонн |
| `deal_type` | `enum` (`Formula` \| `SPOT`) | да | Тип сделки: формульный контракт или спот без долгосрочной формулы |
| `currency` | `string` | да | Валюта строки (ISO-код, напр. `RUB`, `USD`) |
| `country` | `string` | нет | Страна направления поставки |
| `region` | `string` | нет | Регион / область |

**Примечание.** `deal_type` в прогнозе — бизнес-тип сделки (`Formula` / `SPOT`). Не путать с полем `deal_type` в каталоге формул (классификатор SAP: `A1`, `1`, `2`, …).

## 2. Формула (`Formula`)

Запись каталога договоров: текст выражения и область применения. Одна формула «накрывает» все подходящие строки прогноза.

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| `formula_id` | `string` | да | Идентификатор формулы (напр. `Z900026393`) |
| `formula_text` | `string` | да | Математическое выражение текстом |
| `client_id` | `string` | да | Клиент области применения |
| `material_group_m` | `string` \| `null` | нет | Группа материалов M; если пусто — область без привязки к группе |
| `deal_type` | `string` | нет | Классификатор типа цены SAP (не путать с `deal_type` прогноза) |
| `valid_from` | `date` | да | Начало действия формулы |
| `valid_to` | `date` | да | Окончание действия формулы |
| `inactive` | `boolean` | да | Признак неактивности; неактивные в подбор не входят |
| `document_currency` | `string` | нет | Валюта документа формулы (`ВалютаДокумента`) |
| `createdon` | `datetime` | нет | Дата создания в SAP; при нескольких подходящих формулах для авторасчёта берётся запись с максимальным `createdon` |

**Область применения (условия совпадения со строкой прогноза):**

| Условие | Правило |
| --- | --- |
| Клиент | `Formula.client_id` = `DemandForecastLine.client_id` |
| Группа M | `DemandForecastLine.material_id` входит в `Formula.material_group_m` через справочник групп |
| Период | `DemandForecastLine.period` ∈ [`valid_from` … `valid_to`] |
| Неактивность | `inactive` = false |
| Несколько совпадений | показать все; по умолчанию — max(`createdon`) |
| Просроченная активная формула (MVP) | допускается продление `valid_to` до периода строки с пометкой в результате |

## 3. Компонент формулы (`FormulaComponent`)

Переменная из `formula_text`: константа, котировка, курс и т.д.

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| `formula_id` | `string` | да | Ссылка на формулу |
| `var_name` | `string` | да | Имя переменной в выражении (напр. `MRC`, `L1`, `EX`) |
| `component_type_code` | `string` | да | Код типа терма SAP → `TermType.type_code` |
| `component_type_label` | `string` | нет | Человекочитаемая расшифровка типа |
| `value_demo` | `decimal` \| `null` | нет | Фиксированное значение для констант / надбавок / логистики и т.п. |
| `quote_tech_name` | `string` \| `null` | нет | Техническое имя котировки или код курса (для типов «котировка» / «курс») |
| `quote_type` | `string` \| `null` | нет | Справочный вид из SAP (не равен каскаду подстановки Факт→ОФ→ППР) |
| `period_rule` | `string` \| `null` | нет | Правило выбора периода значения |

**Основные коды `component_type_code` и источник значения:**

| Код | Смысл | Откуда берётся значение |
| --- | --- | --- |
| `1` | Котировка | `quotes` через `quote_mapping` |
| `5` | Курс валют | `currency_rates` |
| `H` | Константа | `value_demo` |
| `A` | Надбавка | `value_demo` |
| `B` | Логистика | `value_demo` |
| `C` | Котировочная корректировка | `value_demo` |
| `D` | Скидка | `value_demo` |
| `E` | Прочее | `value_demo` |
| `7` | Прайс-лист (CONDROW) | контур Pricing / SAP |
| `0`, `6` | Группировки | агрегаты расчёта Pricing |

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| `type_code` | `string` | да | Код типа (`1`, `5`, `H`, `B`, …) |
| `type_label` | `string` | да | Название типа |
| `category` | `string` | нет | Категория: котировка / курс / константа / прайс-лист / группировка |
| `fixed_term_kind` | `string` \| `null` | нет | Вид фиксированного терма (Константа, Логистика, Надбавка, …) |

## 5. Маппинг котировок (`QuoteMapping`)

Связывает имя котировки в формуле с кодом в справочнике котировок.

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| `quote_tech_name` | `string` | да | Имя котировки в SAP / компонентах (`Имя котировки`) |
| `quote_code` | `string` \| `null` | нет | ID в озере данных → `Quote.quote_code` (`ID в озере данных`) |
| `quote_kind` | `string` \| `null` | нет | Вид котировки |
| `quote_currency` | `string` \| `null` | нет | Валюта котировки |

**Связь:** `FormulaComponent.quote_tech_name` = `QuoteMapping.quote_tech_name` → `Quote.quote_code`.

---

## 6. Котировка (`Quote`)

Рыночная цена на конкретную дату и тип публикации.

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| `quote_type` | `enum` / `string` | да | Тип публикации: `Факт`, `ОФ`, `ППР` (и др.) |
| `quote_name` | `string` | нет | Отображаемое имя |
| `tech_quote_name` | `string` | нет | Техническое имя |
| `quote_code` | `string` | да | Код котировки в озере данных |
| `quote_date` | `date` | да | Дата котировки |
| `quote_currency` | `string` | нет | Валюта значения |
| `quote_val` | `decimal` | да | Числовое значение |
| `tech_load_ts` | `datetime` | нет | Техническая метка загрузки |

**Правило выбора при нескольких типах на дату:** приоритет **Факт → ОФ → ППР**. Отсутствие значения на нужную дату — ошибка по строке расчёта, не подстановка нуля.

## 7. Курс валюты (`CurrencyRate`)

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| `version_type` | `enum` / `string` | да | Тип версии: `Факт`, `ОФ`, `ППР` |
| `currency_name` | `string` | да | Код / имя валютной пары (напр. `CNY`) |
| `calmonth` | `date` | нет | Календарный месяц |
| `calday` | `date` | да | Календарный день курса |
| `currency_value` | `decimal` | да | Значение курса |

**Правило выбора:** каскад `version_type` **Факт → ОФ → ППР**, затем ближайшая `calday`. Нет курса — ошибка строки.

## 8. Группа материалов (`MaterialGroupMembership`)

Связь материала с группой M; нужна, чтобы формула «на группу» покрывала все входящие SKU.

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| `group_m` | `string` | да | Код группы M (напр. `MT00000116`) |
| `material_id` | `string` | да | Материал, входящий в группу |
| `material_name` | `string` | нет | Наименование материала |
| `name_lvl_2` | `string` \| `null` | нет | Название уровня иерархии 2 |
| `name_lvl_3` | `string` \| `null` | нет | Название уровня иерархии 3 |
| `level` | `string` | нет | Уровень в иерархии |

## Связи между сущностями

| От | К | Тип связи | Ключ / условие |
| --- | --- | --- | --- |
| Формула | Компонент формулы | 1 : N | `formula_id` |
| Компонент формулы | Тип терма | N : 1 | `component_type_code` → `type_code` |
| Компонент формулы | Маппинг котировок | N : 1 | `quote_tech_name` → `Имя котировки` (для типа «котировка») |
| Маппинг котировок | Котировка | 1 : N | `quote_code` |
| Компонент формулы | Курс валюты | N : N (по правилу) | `quote_tech_name` / имя пары + дата периода + `version_type` |
| Формула | Группа материалов | N : 1 | `material_group_m` → `group_m` |
| Группа материалов | Строка прогноза | 1 : N | `material_id` |
| Формула | Строка прогноза | N : M | область применения: клиент + группа M + период (+ активность) |

### Схема потока (логическая)

```text
Формула ──< Компонент формулы >── Тип терма
   │                │
   │                ├──(котировка)──> Маппинг ──> Котировка (дата, тип)
   │                └──(курс)───────> Курс валюты (день, version_type)
   │
   ├── material_group_m ──> Группа материалов ──> material_id
   │
   └── область применения ──> Строки прогноза спроса
```

### Цепочка расчёта одной строки

1. Взять **формулу** (клиент + группа M + окно `valid_from`…`valid_to`).
2. Через **группы материалов** развернуть состав группы в список `material_id`.
3. Найти **строки прогноза**, у которых совпали клиент, период и материал из группы.
4. По **компонентам** подтянуть константы, **котировки** (через маппинг) и **курсы**.
5. Вычислить `formula_text` → итоговая цена (и расшифровка).

## Сущности результата (задел на следующие этапы)

На этапе 1 достаточно спроектировать хранение исходников. Для MVP расчёта понадобятся производные объекты (детали — на следующих этапах):

| Сущность | Назначение (кратко) |
| --- | --- |
| `CalculationRun` | Прогон расчёта: период, параметры, время, статус |
| `CalculationResult` | Результат по `row_id`: выбранная `formula_id`, цена, статус, признак продления срока |
| `FormulaCandidate` | Список всех формул, подошедших к строке |
| `ComponentResolution` | Подставленное значение каждого `var_name` и источник |
| `ManualOverride` | Ручная цена / принудительный выбор формулы пользователем |

## Ограничения этапа 1

- Модель опирается на обезличенный учебный срез; эталонные ответы цен в материалы не входят.
- Договор и доп. соглашение в выгрузках явно не представлены — в MVP достаточно каталога формул как носителя правил цены.
- Правила продления просроченных формул и каскад Факт→ОФ→ППР относятся к логике подбора/расчёта; в схеме данных для них достаточно полей `valid_*`, `inactive`, `createdon`, `quote_type` / `version_type`.
