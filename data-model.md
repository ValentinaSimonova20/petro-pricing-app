# Модель данных (этап 1)

Документ построен по **фактическому составу** восьми Excel-файлов, встроенных в исходную презентацию. Имена колонок, типы и ключи — как в файлах (лист один на файл). Описание в HTML-презентации **не использовалось** как источник схемы.

Файлы для сверки: `data/raw/*.xlsx` (извлечены из презентации).

Типы — логические, по содержимому ячеек (openpyxl, `data_only`).

## Связи между сущностями

Имена колонок — **как в Excel**.

| От | К | Как связать |
| --- | --- | --- |
| `formulas` | `formula_components` | `Ключ формулы` = `Формула` (1 : N, полное покрытие) |
| `formula_components` | `term_types` | `Тип фиксированного терма` = `type_code` (N : 1) |
| `formulas` | `ssp` | `client_id` = `client_id` (N : M по клиенту) |
| `formulas` | `material_groups` | `Подгруппа материалов` = `hname` (N : 1, покрытие неполное) |
| `formulas` | `ssp` / `material_groups` | `Материал` ↔ `mtr_nsi_code` / `code_nsi` |
| `ssp` | `material_groups` | `mtr_nsi_code` (как строка) = `code_nsi` |
| `formula_components` | `quote_mapping` | `Имя котировки` (+ при необходимости `Происхождение котировки`) |
| `quote_mapping` | `quotes` | `ID в озере данных` = `quote_code` |
| `formula_components` | `currency_rates` | через `Имя котировки` / правила периода и `currency_name` + `calday` (тип терма `5`) |

### Логическая схема

```text
ssp ─────────────────────────────── formulas
 │  client_id                         │ Ключ формулы
 │  mtr_nsi_code                      │
 │                                    ▼
 │                          formula_components
 │                             │         │
 │                             │         ├── Тип фиксированного терма → term_types
 │                             │         ├── Имя котировки → quote_mapping → quotes
 │                             │         └── (тип 5) → currency_rates
 │                             │
 └─ mtr_nsi_code ── code_nsi ── material_groups ◄── Подгруппа материалов (hname)
```

## Отличия от упрощённых имён в презентации

Ниже — чтобы не опираться на «красивые» алиасы из HTML:

| Было в прежних описаниях | Факт в Excel |
| --- | --- |
| `deal_type` / `SPOT` | колонка `contract`, значения `Formula` / `Spot` |
| `material_id`, `material_name` | `mtr_nsi_code`, `mtr_nsi_name` |
| `forecast_tons` | `forecast` |
| `formula_id`, `formula_text` | `Ключ формулы`, `Формула` |
| `material_group_m` | `Подгруппа материалов` |
| `valid_from` / `valid_to` / `createdon` / `inactive` | `"Действительно с"…`, `Метка времени "Действительно по"`, `Дата создания`, `Неактивна` |
| `var_name`, `component_type_code`, `value_demo`, `quote_tech_name` | `Имя переменной`, `Тип фиксированного терма`, `Значение`, `Имя котировки` |
| `group_m`, `material_id` в группах | `hname`, `code_nsi` (+ `V_MTR_name`, уровни, `datuv`/`datub`/`leaf`) |

Модель данных этапа 1 фиксирует **имена из файлов** как канонические.
